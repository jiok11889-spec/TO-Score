## 매달 대회 후 작업 순서

### 1단계 — 스코어 이미지 확인

대회 종료 후 사용자가 `캡쳐/YYMMDD/` 폴더에 이미지를 넣어준다.  
예: `캡쳐/260612/KakaoTalk_20260612_133934177.jpg`

이미지에서 읽어야 할 것: **스트로크(타수) 점수** (신페리오 점수 아님, 핸디 미적용 실타수)

- 대회 결과 이미지에 "스트로크" 컬럼이 따로 표시된다
- 이미지 하단에 전체 순위가 없으면 두 장에 걸쳐 있을 수 있다
- **비회원/게스트 제외**: TIER_MAP에 없고 모임 정식 멤버가 아닌 사람은 스코어에서 뺀다 (예: 최지훈 같은 경우)
- 이미지가 흐려서 못 읽겠으면 사용자에게 텍스트로 요청한다

### 2단계 — update_scores.py 수정

`update_scores.py` 상단 변수 4곳을 수정한다.

#### 2-1. 스코어 딕셔너리 교체

변수 이름도 대회에 맞게 변경 (예: `JUN_2026_SCORES`, `JUL_2026_SCORES`).  
주석에 대회 날짜와 참가 인원 수 기재.

```python
# 26년 7월 XX일 대회 스코어 (N명)
JUL_2026_SCORES = {
    "이름": 타수,
    ...
}
```

딕셔너리 참조 부분(2곳)도 함께 수정:
- `if name in JUL_2026_SCORES:` (score_updates)
- `score_missing = [n for n in JUL_2026_SCORES ...]`

#### 2-2. 이벤트 메타 변수

```python
EVENT_COL_NAME = "26' 5차 대회"   # 대회 회차 증가
EVENT_DATE     = "7/XX/2026"      # 실제 날짜
TIER_ROUND_COL = "8회"            # 7회 다음
TIER_ROUND_DATE = "7/XX/2026"
```

**회차 규칙**: 대회 회차(N차)와 티어 히스토리 회차(N회)는 별개.  
현재 기준: 4차 대회 = 7회 티어 기록. 다음 대회는 `TIER_ROUND_COL = "8회"`.

### 3단계 — 티어 자동 조정

#### 기본 원칙

- `config.py`의 `PRO_PLAYERS` (김산, 이현우, 이형석, 임승언, 윤석원)는 **항상 T0 고정**, 실력에 상관없이 티어 불변
- T0이지만 PRO가 아닌 선수(나한영, 홍경택)는 조정 대상
- 티어 변동은 **1단계씩만** (T2→T0 같은 2단계 이동 없음)
- 대회 미참가자는 티어 변동 없음

#### 조정 알고리즘

대회 참가자를 현재 티어별로 그룹화한 뒤, **티어 경계선마다** 비교한다.

**승격 조건 (T_n → T_(n-1)):**  
T_n 참가자 중 **T_(n-1) 참가자 대부분보다 낮은 타수(좋은 성적)** 를 기록하면 승격.  
또는 T_n 그룹 내 최상위 성적이 T_(n-1) 하위권보다 확실히 좋으면 승격.

**강등 조건 (T_n → T_(n+1)):**  
T_n 참가자 중 **T_(n+1) 참가자보다 높은 타수(나쁜 성적)** 를 기록하면 강등.  
또는 T_n 그룹 내 최하위 성적이 T_(n+1) 상위권보다 확실히 나쁘면 강등.

#### 실전 적용 예시 (4차 대회 6/12 기준)

| 참가자 | 현재 티어 | 타수 | 판단 | 결과 |
|--------|-----------|------|------|------|
| 이은광 | T2 | 81 | T1 김수민(81)과 동점, T1 전원보다 좋거나 같음 | **T2→T1 승격** |
| 임익준 | T1 | 94 | T3 김준기(94)와 동점, T2 전원보다 나쁨 | **T1→T2 강등** |
| 박헌우 | T4 | 93 | T3 김준기(94)보다 좋음 | **T4→T3 승격** |
| 김준기 | T3 | 94 | T4 박헌우(93)에게 밀림 | **T3→T4 강등** |
| 조성태 | T2 | 96 | T3 김준기(94)보다 나쁨, T2 최하위 | **T2→T3 강등** |

#### 판단이 애매한 경우

- 참가자 수가 1~2명뿐인 티어: 인접 티어와 직접 비교해서 판단
- 성적 차이가 1~2타 이내 경계선: 보수적으로 유지 (변동 없음)
- 확신이 없으면 사용자에게 제안하고 확인 받은 후 반영

### 4단계 — 스크립트 실행

```powershell
python update_scores.py
```

정상 출력 확인:
- `[OK] 스코어 입력: N명` — 참가자 수와 일치하는지 확인
- `[OK] 티어 업데이트: 41명` — 전체 멤버 티어 갱신
- `[!!] 스코어 미입력` — 이름 불일치 시 출력. 시트 이름과 다르면 수정

---

### 5단계 — 커밋 & Render 배포

```powershell
git add update_scores.py
git commit -m "feat: 26' N차 대회(M/DD) 스코어 및 티어 업데이트

상승: 이름(T2→T1), ...
하락: 이름(T1→T2), ..."

git push origin main
```

push 하면 Render 자동 배포. 대시보드는 서버가 Sheets 데이터를 실시간으로 읽으므로 별도 빌드 불필요.
